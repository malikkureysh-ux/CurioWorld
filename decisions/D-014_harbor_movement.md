# D-014 — Vertical-Slice-Bewegung: Kran-Schwingen als Highlight

**Status:** offen (Validierung in Phase 4)
**Datum:** 2026-06-27
**Owner:** Lead Designer + Tech Lead

## Problem

Welche bezirks-spezifische Bewegung wird in Hamburg Harbor erprobt?

## Optionen

### Option A — Kran-Schwingen

- Spieler:in steuert einen Kran, schwingt sich an die nächste Plattform.
- Physikalisch lehrreich (Hebelgesetz, Schwingungen).

### Option B — Boot-Gleitbahnen

- Spieler:in gleitet auf Wasserstraßen.
- Sanfter, weniger physikalisch.

### Option C — Standard-Springen

- Klassische Parkour-Bewegung ohne Spezial-Mechanik.

### Option D — Mehrere kombiniert

- Kran-Schwingen als Highlight + andere Mechaniken.

## Bewertungsmaßstab

- Spielspaß
- Lernanbindung
- Performance
- Einzigartigkeit der Bezirks-Identität

## Entscheidung

**Option D — Mehrere kombiniert (Kran-Schwingen als Highlight).**

## Begründung

- **Vielfalt:** Mehrere Bewegungs-Mechaniken erhöhen Wiederspielbarkeit.
- **Kran-Schwingen als Highlight:** Einzigartig, physikalisch lehrreich, starke Mechanik-Identität.
- **Andere Mechaniken:** Boot-Gleitbahnen, Wandläufe an Speichern, Standard-Parkour für Abwechslung.

## Erwartete Wirkung

- Starke Mechanik-Identität für Hamburg Harbor.
- Spieler:innen-Motivation durch Vielfalt.
- Klarer Showcase für Physik-Lernen.

## Risiken

- **R-1:** Komplexität der Steuerung.
  - Mitigation: schrittweiser Tutorial-Aufbau, adaptive Hilfe.
- **R-2:** Performance-Budgets könnten überschritten werden.
  - Mitigation: frühzeitige Performance-Tests in Phase 4.

## Validierungsplan

| Phase | Aktion |
|---|---|
| Phase 4 | Playtest mit 30 Spieler:innen pro Altersband |
| Phase 4 | Performance-Tests auf Low-End-Mobile |
